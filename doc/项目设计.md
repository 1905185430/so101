# SO-101 快速部署工具链 — 项目设计

## 1. 背景与目标

SO-101 双臂遥操作项目原散落在多个独立脚本中，换电脑后需要手动改端口、改设备号，部署效率低。本项目将所有硬件配置抽取为统一的 YAML 文件，配合 CLI 工具链，实现新电脑"一行命令"即可启动遥操作。

### 3.2 设备标识策略

- **摄像头**：优先使用 `/dev/v4l/by-id/...` 路径，物理插拔后路径稳定
- **机械臂**：使用 `/dev/serial/by-id/...` 路径，基于 USB Serial ID（CH9101F 芯片）
- **不推荐**：`/dev/videoN`、`/dev/ttyACM0` 等随插拔变化的路径

## 4. CLI 命令

| 命令 | 功能 |
|---|---|
| `so101 list` | 只读探测系统当前设备（不修改配置） |
| `so101 scan [--all]` | 探测设备并注册到 camera_config.yaml |
| `so101 check` | 检查所有场景配置在当前系统是否可用 |
| `so101 calibrate --arm <name>` | 校准指定机械臂 |
| `so101 teleop [--mode 1to1]` | 启动遥操作（三种模式） |
| `so101 record --scene <name>` | 数据录制 |
| `so101 capture [--role top]` | 摄像头预览与截图 |

## 5. 遥操作三种模式

- **1to1**：单主臂控制对应单从臂（一对一）
- **1toN**：单主臂同步控制双从臂（适合双手协作任务）
- **dual**：双主臂独立控制双从臂（双臂协调任务）

## 6. 部署流程（新电脑）

```bash
# Step 1: 克隆项目
git clone https://github.com/1905185430/lerobot_workdocs.git ~/lerobot_workdocs
cd ~/lerobot_workdocs/so101

# Step 2: 一键安装（创建 conda 环境、安装依赖、配置权限）
bash scripts/setup.sh
conda activate lerobot

# Step 3: 连接硬件后，扫描注册
so101 scan --all

# Step 4: 检查场景
so101 check

# Step 5: 首次校准（如需要）
so101 calibrate --arm follower_left
so101 calibrate --arm leader_left

# Step 6: 启动遥操作
so101 teleop --mode 1to1 --arm left
```

## 7. 已知限制

- 机械臂波特率固定 500000，需要 feetech 驱动支持（setup.sh 自动处理）
- 校准数据保存在 `~/.lerobot/calibration/`（LeRobot 默认路径）
- 双臂遥操作需要 4 个臂同时在线
- 仅支持 Ubuntu 22.04

## 8. 依赖

- Python ≥ 3.10
- conda（推荐）或 pip
- LeRobot ≥ 0.5.1（lerobot conda 环境）
- opencv-python, pyyaml
